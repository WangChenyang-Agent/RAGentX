# Go 并发编程

## Goroutine

Goroutine 是 Go 语言轻量级的线程实现，由 Go 运行时管理。

```go
go func() {
    fmt.Println("Hello from goroutine")
}()
```

### Goroutine 特点

1. **轻量级**：初始栈大小只有 2KB
2. **快速创建**：创建一个 goroutine 只需要几微秒
3. **调度高效**：使用 GMP 调度模型

## Channel

Channel 是 goroutine 之间通信的主要方式，遵循 "不要通过共享内存来通信，而要通过通信来共享内存" 的理念。

```go
ch := make(chan int)
ch <- 1  // 发送
x := <-ch  // 接收
```

### Channel 类型

1. **无缓冲 channel**：同步通信，发送和接收必须同时进行
2. **有缓冲 channel**：异步通信，缓冲区满时才会阻塞

```go
// 无缓冲
ch1 := make(chan int)

// 有缓冲
ch2 := make(chan int, 10)
```

## Select

Select 用于同时监听多个 channel 操作：

```go
select {
case msg := <-ch1:
    fmt.Println("Received from ch1:", msg)
case ch2 <- value:
    fmt.Println("Sent to ch2")
default:
    fmt.Println("No channel ready")
}
```

## 常见并发问题

1. **死锁**：所有 goroutine 都被阻塞
2. **竞态条件**：多个 goroutine 同时访问共享资源
3. **goroutine 泄漏**：goroutine 无法退出导致资源泄漏

## Mutex

互斥锁用于保护共享资源：

```go
var mu sync.Mutex
var count int

func increment() {
    mu.Lock()
    defer mu.Unlock()
    count++
}
```

## WaitGroup

WaitGroup 用于等待一组 goroutine 完成：

```go
var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        fmt.Println("Worker")
    }()
}
wg.Wait()
```
