# Go 接口与反射

## 接口（Interface）

接口定义了一组方法签名，任何实现了这些方法的类型都自动实现了该接口。

```go
type Writer interface {
    Write([]byte) (int, error)
}

type File struct{}

func (f *File) Write(data []byte) (int, error) {
    return len(data), nil
}
```

### 空接口

空接口 `interface{}` 不包含任何方法，所有类型都实现了空接口：

```go
var anything interface{}
anything = 42
anything = "hello"
anything = []int{1, 2, 3}
```

### 接口组合

```go
type ReadWriter interface {
    Reader
    Writer
}
```

### 类型断言

```go
var x interface{} = "hello"
if s, ok := x.(string); ok {
    fmt.Println(s)
}
```

### 类型选择

```go
switch v := x.(type) {
case int:
    fmt.Println("int:", v)
case string:
    fmt.Println("string:", v)
default:
    fmt.Println("unknown")
}
```

## 反射（Reflection）

反射允许程序在运行时检查类型信息和操作值。

### reflect.Type

获取类型信息：

```go
t := reflect.TypeOf(42)
fmt.Println(t.Name())  // int
fmt.Println(t.Kind())  // int
```

### reflect.Value

获取值信息：

```go
v := reflect.ValueOf(42)
fmt.Println(v.Int())  // 42
```

### 修改值

```go
x := 42
v := reflect.ValueOf(&x).Elem()
v.SetInt(100)
fmt.Println(x)  // 100
```

### 调用方法

```go
v := reflect.ValueOf(obj)
method := v.MethodByName("DoSomething")
method.Call(nil)
```

## 反射的应用场景

1. **序列化/反序列化**：JSON、XML 等
2. **ORM 框架**：数据库映射
3. **依赖注入**：自动装配
4. **通用工具**：如 fmt.Printf

## 性能考虑

反射比直接调用慢，应避免在性能关键路径使用。
